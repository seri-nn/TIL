## Lecture 01, 02
수업 내용 요약 : 프로그래밍의 전반적인 정의와 c언어의 개념과 특징, 가장 기초적인 c언어 실습

### 프로그램과 프로그래밍 언어
- 프로그램(Program) : 어떤 문제를 해결하기 위해 그 처리 방법과 순서를 기술하여 컴퓨터에 주어지는 일련의 **명령문 집합체**
- 프로그래밍 언어 : 프로그램을 개발하기 위해 사용하는 언어, 사람-컴퓨터 서로 의사교환을 하기 위한 언어, 사람이 컴퓨터에게 지시할 명령어를 기술하기 위해 만들어진 언어

### 컴파일러와 기계어
- 컴파일러(Compiler) : 프로그래밍 언어로 작성된 프로그램을 **컴퓨터가 이해할 수 있는 기계어로 번역** 하는 것 (compile : 번역하다) 
- 기계어(Machine Language) : 컴퓨터가 이해할 수 있는 **0과 1로 구성된 언어**

### C언어의 개론적 이야기
- C언어 : 유닉스(Unix) 운영체제 개발을 위해 데니스 리지와 켄 톰슨이 설계한 범용적인 고급 언어
- C언어 등장 이전의 유닉스 개발
  - 어셈블리 언어(Assembly Language)라는 저급 언어로 만들어짐
  - 어셈블리 언어는 하드웨어에 따라서 구성이 달라지기 때문에 CPU 별로 유닉스를 각각 개발해야 했음
- C언어 등장 이후의 유닉스 개발
  - C언어의 구성은 CPU에 따라 나뉘지 않기 때문에 유닉스를 각각 개발할 필요가 없게 됨
 
- 어셈블리 언어(Assembly Language) : 기계어를 사람이 이해하기 쉬운 기호 형태로 만든 용어 -> 기계어보다 프로그래밍이 용이
- 고급 언어(High-level) : 사람이 이해하기 쉬운 언어

  ex. 포트란, c, c++ ...etc

- 저급 언어(Low-level) : 기계어에 가까운 언어

  ex. 어셈블리어, 기계어 

### 컴파일러와 어셈블러
- 컴파일러(Compiler) : 고급 언어로 작성된 프로그램을 기계어 또는 목적코드(Object Code)로 바꿔주는 프로그램
- 어셈블러(Assembler) : 어셈블리 언어로 작성된 프로그램을 기계어로 바꿔주는 프로그램

컴파일러와 어셈블러 둘다 프로그래밍 언어를 기계어로 바꾸어 주지만 컴파일러는 고급 언어를 어셈블러는 오로지 어셈블리 언어만 번역 가능
### C언어의 장점
- 절차지향적 특성 지님(인간이 사고하는 방식과 유사)
- C언어로 작성된 프로그램은 이식성이 좋음
  - CPU에 따라 프로그램을 재작성할 필요 없음
- C언어로 구현된 프로그램은 좋은 성능을 보임
  - 메모리의 사용량을 줄일 수 있음

### C프로그램의 완성 과정
프로그램의 작성 -> 작성한 프로그램의 컴파일 -> 컴파일 된 결과물의 링크 -> 실행파일 생성(.exe)

- 링크(Link) : 컴파일로 이미 만들어진 기계어를 라이브러리와 연결시켜 실행 가능한 파일로 만드는 것


### C언어 코드 알아보기 
- 소스 코드(Source Code) : 프로그램 제작에 사용되는 설계 파일 확장자(.c)

- C언어의 기본 단위는 함수이다.
- **main : 제일 먼저 호출되는 함수**
  - 반드시 main이라는 이름의 함수를 정의해야만이 프로그램을 실행시킬 수 있음 단, 하나의 project에서 main이라는 이름을 또 사용할 수 는 없음?

- 함수의 기능 : 함수의 기능은 중괄호 안에 표현이 됨

- ; : 함수 내에 존재하는 문장의 끝에는 세미콜론을 붙여야 됨 -> 하나의 명령어가 끝남을 표현하기 위해서. 줄 바뀜은 문장의 바뀜을 뜻하는 것이 아님

### Hello.c 소스 코드 분석
```cpp
# include <stdio.h>
int main(void)
{
  printf("Hello world\n");
  return 0;
}
```
- printf : 입력받은 인자 값을 모니터에 출력
  - c프로그래밍에서 기본적으로 제공되는 함수인 표준 라이브러리(Standard Library)임
  - 출력하고 싶은 문자열을 " "로 감싸야함. (파이썬과 달리 ' '로 감싸면 오류 발생)

- %d\n : 정수의 형태로 출력하라는 서식 문자 cf) d : demical, 10진법의
  - %d에 들어갈 정수는 , 뒤에 입력하고 d의 개수와 정수의 개수가 일치해야함
  - 정수 즉, 숫자이기 때문에 사칙연산이 가능함. %d가 아닌 그냥 입력한 수는 문자열로 인식함 
    ex. printf("%d-%d\n, 10,20);
    ex. printf("%d\n", 311 + 20); 

- \n : 줄 바꿈

- 헤더 파일(Header file) : 확장자가 .h로 끝나는 파일
  - stdio.h(standard input/output library) 즉, 표준 라이브러리를 헤더 파일로 불러야지 표준 라이브러리 함수를 사용할 수 있음
  - 경우에 따라 하나의 프로그램 내에 여러 개의 헤더 파일이 포함되기도 함
  - **헤더 파일의 포함을 알리는 선언이 제일 먼저 등장해야 됨 ex. # include<stdio.h>**
 
- return : 함수를 빠져나옴(return;만 입력시) / 함수를 호출한 영역으로 값을 반환(return 0; 처럼 return 뒤에 값을 입력시)
  - main 함수가 최종적으로 반환해 주는 값을 통해 프로그램이 잘 종료되었는지 문제가 발생했는지 파악
  - return 0;을 약속이라 생각하면 편함
 
- 주석(Commet) : 소스코드에 삽입된 메모. 컴파일의 대상에서 제외가 되기 때문에 실행 결과에는 영향을 미치지 않음
    - 단축키 : 주석 설정 ctrl k c / 주석 해제 ctrl k u
    - 블록 단위 주석 /* */
        - 블록 단위 주석은 중첩되는 형태로 작성 불가
    - 행 단위 주석 //
